UConn DX Group Questions

By: John "Jack" Medrek


What solution would you use for mapping? Why? (The DXG has used MapBox before)

I would recommend we stay using MapBox. The other main option is Google Maps API, which while being more feature-rich is also quite a bit more expensive. Because of the scale of our site, it probably makes more sense to go with the cheaper option: MapBox.

We probably can’t afford to make a bunch of native apps (iOS, Android, etc). How would we make this thing cross-platform?

One way to make this app cross-platform would be to make it as a PWA (progressive web app). This would be a website that behaves and looks like a mobile app, and it would provide the option to work off a single codebase. Any device with a browser would be able to use it.

Any front-end frameworks you’d suggest we use? Why?

I would suggest Angular.js as a front-end framework. For starters, it’s very popular and thus has a large amount of corresponding information online to answer any questions we might have while using it. More technically, Angular can use two-way binding to keep the POIs synchronized on the mobile app we create and its corresponding website.

Any back-end technology you’d suggest we use? Why?

If we choose Angular.js as our front-end framework, we can choose almost any back-end technology, the only requirement is that said back-end technology can provide a RESTful API that accepts JSON. Given this, I would suggest using Express.js on the Node.js platform, because that leaves just one language to learn, JavaScript.

How do we handle the login stuff?

While we could build our own login table and encrypt the users’ data ourselves, it would probably be best to use something like OAuth to allow sites like Google or Facebook to handle our users’ authentication. It would be easier and safer to use these systems which have been thoroughly tested than to build our own.

Where should we store the data on uploaded POIs and what does that data look like? (The DXG has used Cloud Firestore and Supabase a lot lately)

While Supabase and Cloud Firestore are good database technologies, I think another way our data could be stored on a database is using MongoDB. This is mainly because of its ease of use with the previously mentioned front-end and back-end technologies. MongoDB, along with Angular.js and Express.js, forms the MEAN stack — an all-JS web framework.

That data would be in the form of an image file, a string containing the title/name of the POI, and a string containing the POI’s description.

What happens if the researchers lose cell service in the middle of the state forest?

If we use a PWA as previously mentioned, the only point when our PWA would require an Internet connection is when it’s being installed or synced to the website. If it’s permissible to wait until there is cell service to update the website, then a PWA should not require cell service otherwise.
